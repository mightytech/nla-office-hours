# Content Index

Essays, white papers, and working code exploring what software looks like when the documentation is the application — when prose is the source code and an AI that reads it is the runtime. The ideas range from technical architecture to governance to domain applications in dating, tax filing, policy simulation, and personal productivity. Everything described here is either built and open source, or illustrated in enough detail to evaluate.

## Reading Paths

**"What are NLAs?"** — Start with the probe report. It explains the paradigm through five findings, grounded in things actually built.

**"Is this just prompt engineering?"** — Read [Adjacent Approaches](adjacent-approaches-to-nla.md), then the probe report for the full picture.

**"What does this look like applied to a real domain?"** — Pick whichever domain exploration interests you. [NLA Dating](nla-dating-app.md) is the most complete convergence; [Chief of Staff](nla-chief-of-staff.md) is the most universally relatable; [Tax Filing](nla-tax-filing-1040ez.md) is the clearest hybrid model illustration.

**"What about governance and values?"** — Start with [Executable Governance](executable-governance.md), then read [NLA Writer Values](nla-writer-values.md) to see a working values document.

**"Show me something surprising."** — [Absurd Things That Work](absurd-things-that-work.md).
